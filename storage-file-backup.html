System Restore

Instructions for use-

  • This operation will reset the system data, non-operation and maintenance or related technical personnel should not operate!
  • By writing the backup database to the new database and setting it as the system default, data recovery is achieved. The new database configuration parameters will be appended to the system configuration file config/setting_user.php. If the system is abnormal after the recovery is executed, the appended part of the file can be removed without affecting the previous system data.
  • Note: When the database type is MySQL, a new library (original library name_current date_rebuild) will be created based on the current configuration information. Non-root users may not have sufficient permissions, so you need to set permissions for the user first.
  • For example, the current database configuration information is: user: kod; password: kod123. Use the root account to log in to the database and execute the corresponding SQL statement to set the permissions (the permissions can be revoked after the test is passed and the recovery is successful).
  • Setting permissions:grant all privileges on *.* to 'kod'@'localhost' identified by 'kod123' with grant option;flush privileges; Revoke permissions:revoke all on *.* from 'kod'@'localhost';flush privileges;
